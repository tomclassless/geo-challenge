# 地理大挑戰 🗺️

給暑假小朋友輪流玩的地理題庫 App。一台 iPad、20 人輪流;每週換一個地區題庫;
玩完看每題正確率與每人對錯;成績存雲端可事後追蹤。**在家連網同步、現場離線也能玩。**

---

## 使用模型(很重要)

- **在家(有網路)**:打開 App 按「🔄 同步」→ 下載最新題庫、上傳之前累積的成績。
- **首次安裝**:在家用 Safari 開一次網址,點分享 →「加入主畫面」。之後 App + 題庫 + 媒體都存在平板裡。
- **現場(沒網路)**:從主畫面圖示開 App,完全離線就能玩,成績暫存平板。
- **回家**:再按一次「同步」把成績傳上雲端。

> ⚠️ 第一次一定要在家連網開啟並同步一次,才能離線使用。

---

## 一、建立 Google 試算表(題庫 + 名單 + 設定 + 成績)

新建一個 Google 試算表,建立以下分頁:

### 地區分頁(每個地區一個,分頁名 = 地區名,例如「桃園」「台北」)
第一列為標題,欄位如下:

| id | type | question | media | optA | optB | optC | optD | answer | explain |
|----|------|----------|-------|------|------|------|------|--------|---------|
| 1 | text | 桃園的市花是? | | 桃花 | 杜鵑 | 茶花 | 油桐花 | A | 桃園市花為桃花 |
| 2 | image | 圖中是哪個機場? | taoyuan.jpg | 桃園機場 | 松山機場 | 小港機場 | 台中機場 | A | |

- `type`:`text` / `image` / `video`
- `media`:圖片或影片來源。
  - **圖片檔名**(建議小寫英數,如 `taoyuan.jpg`)→ 在 App 首頁「題目圖片管理」上傳同名圖片(存到老師的 Google 雲端硬碟),同步後離線也看得到。不需要再手動把檔案丟進 `public/media/`。
  - **完整網址**(http 開頭)→ 第一次連網看過後會自動快取。
  - **影片**請用 **YouTube 連結**(`type` 填 `video`),只在連網時播放。
- `answer`:正解選項字母 `A`/`B`/`C`/`D`。選項可只填 2–4 個。

### `Players` 分頁(名單)
| name |
|------|
| 小明 |
| 小華 |

### `Config` 分頁(設定)
| key | value |
|-----|-------|
| teacherPin | 1234 |
| timerSeconds | 30 |

### `Results` 分頁
不用自己建,第一次上傳成績時會自動建立。

## 二、部署後端(Google Apps Script)

1. 試算表 →「擴充功能」→「Apps Script」。
2. 把 [`apps-script/Code.gs`](apps-script/Code.gs) 內容整段貼進去,存檔。
3. 右上「部署」→「新增部署作業」→ 類型「網頁應用程式」:
   - 執行身分:**我**
   - 誰可以存取:**任何人**
4. 複製產生的網址(`…/exec` 結尾)。

---

## 三、部署前端(這個 App)

```bash
npm install
npm run build      # 產出 dist/
npm run preview    # 本機預覽
```

把 `dist/` 上傳到任何靜態主機(GitHub Pages / Netlify 皆可、免費)。

第一次在 iPad Safari 打開後:
1. 點右下「⚙️ 設定」→ 貼上 Apps Script 的 `…/exec` 網址 → 儲存。
2. 按「🔄 同步」下載題庫與名單。
3. 分享 →「加入主畫面」。

---

## 四、玩法

1. 首頁選本週地區 →「開始輪流作答」。
2. 每位小孩點自己名字 → 限時作答(題目與選項順序會打亂、作答時不顯示對錯)。
3. 個人結算頁只告訴他「答對幾題」+ 是否比上一局進步(同地區重玩時)。
4. 全部玩完 →「📊 看報表」(需老師 PIN):看每題全班正確率 + 每題×每人對錯表。
5. 「🔁 用同題庫再玩一局」可當天重玩,成績分場次記錄。
6. 回家按「同步」上傳;「🕑 看歷史」可回顧各場次。

## 開發

- 前端:Vite + React + TypeScript,PWA(`vite-plugin-pwa`)。
- 離線:Service Worker 快取 App 與媒體;IndexedDB(`src/lib/offlineStore.ts`)存題庫、成績與待上傳佇列。
- 後端:Google Apps Script(`apps-script/Code.gs`)讀寫試算表。
- 主要程式:`src/state/gameStore.ts`(流程與計分)、`src/screens/*`(畫面)、`src/lib/*`(資料與工具)。
